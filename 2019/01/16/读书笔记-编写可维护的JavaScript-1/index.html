  • 在方法中的逻辑片段之间

命名

变量名遵守小驼峰（Camel Case）写法，且前缀应为名词，便于与前缀为动词的函数区分。

尽量在变量名中体现值的数据类型。如count、length和size表明数据类型为数字，而name、title和message表明数据类型为字符串。

避免使用foo、bar和tmp等没有意义的命名，不要用这些命名承载其他附加含义。

函数命名前缀一般为动词，以下为动词常见的约定：

动词 含义
can 函数返回一个布尔值
has 函数返回一个布尔值
is 函数返回一个布尔值
get 函数返回一个非布尔值
set 函数用于保存一个值

null的使用

应该使用null的场景

  • 初始化变量，该变量可能赋值为对象
  • 用来与已初始化的变量比较，比较的变量可以是或不是一个对象
  • 当函数的参数类型为对象时，作参数传入
  • 当函数的返回值类型为对象时，作返回值传出

不应该使用null的场景

  • 不应该使用null检查是否传入某个参数
  • 不应该使用null检查一个未初始化的变量

undefined的使用

建议避免在代码中使用undefined，因为这个值容易和返回“undefined”的typeof运算结果混淆，且不管是值为undefined的变量还是未声明的变量，typeof运算结果都是“undefined”。

尽管这段代码中person和foo都会导致typeof返回“undefined”，但是它们在其他场景中的行为有明显不同（在语句中使用foo会报错，而使用person则不会报错）。

通过禁用特殊值undefined，可以有效确保只在一种情况下typeof运算的结果为“undefined”：当变量未被声明时。如果使用一个可能会（或可能不会）被赋值为对象的变量，则将其赋值为null。

将变量初始值赋为null表明了这个变量的意图——它最终可能赋值为对象。typeof运算符运算null会返回“object”，这样就可以和undefined区分开了。
