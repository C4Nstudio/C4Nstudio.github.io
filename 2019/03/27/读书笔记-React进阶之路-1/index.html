[读书笔记]React进阶之路(1)

1.1 React简介

React的特点

声明式的视图层

React推荐使用JSX来声明视图层，可在视图层中任意使用状态数据。
相比与一般HTML模板区别在于：
JSX的变量作用域（scope）与js的为同一个
JSX可以内嵌编程语言（简单的表达式则都可以）

简单的更新流程

React会自动根据UI状态渲染出最新内容，且状态到UI的数据流单向流动也让组件的更新流程更加清晰。
区别于Angular的双向绑定（界面的操作能实时反映到数据，数据的变更能实时展现到界面），React在界面发生变化时按照配置规则更新相应数据，然后再根据配置规则更新页面状态。

灵活的渲染实现

React不直接渲染视图至终端，而是先渲染成虚拟DOM。
虚拟DOM：
一种在内存中描述DOM树状态的数据结构。当状态发生变化时，React重新渲染虚拟DOM，比较计算之后再给真实DOM做修改。

高效的DOM操作

基于虚拟DOM操作，React减少了对真实DOM的渲染次数和改变内容，提升了效率。

1.2 ECMAScript 6简介

现有标准

ECMAScript 6作为新一代标准，而日常编码过程中使用的无需额外编译的js标准其实也叫做ECMAScript 3（定稿在1999——2000年）

新增内容

除了书中提及的“let、const”、“箭头函数”、“模板字符串”、“结构赋值”、“rest参数”、“扩展运算符”、“class”(在React 16.8版本的proposal中已推荐function写法)和“import、export”外，还丰富了一些内建类型的API，如Array类型增加了find和findIndex方法

1.3 开发环境和工具简介

定义补充

书中将Node.js定义为“一个JavaScript运行时”，原官方说法中的“run-time”可以更进一步解释为运行环境，突出node.js不是一个语言也不是一个框架，是一个用来运行js的环境
